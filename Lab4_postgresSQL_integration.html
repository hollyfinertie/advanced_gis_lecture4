####The plot above clearly indicates potential spatial clustering…. to test for that formally, we are going to implement a spatial-variogram. The first step in the variogram process is to bin the data into distance categories and plot the average semi-variance (gamma) by those bins. This first process creates the R object prop_cor which contains the information most important to us… the range of distance at which points in the prop_sub object are spatially correlated. The range is in decimal degrees, but multiplying it by 69 produces a range in miles. In this case home values are spatially auto-correlated up to about 3 miles away. We can further illustrate this relationship by fitting a model to the binned data using the fit.variogram function. Here we give the function options, from which the best model is chosen. In this case the exponential model (Exp) best fits the data.


prop_cor <- variogram(pr_sq_ft~1, data=prop_sub)
plot(prop_cor)

####Since we know that the data are spatially-autocorrelated, we can feel confident in interpolating the data… but to do so, we must first prepare the data for interpolation. The first thing that I am going to do here is create a new grid which is made up of 5000 points within the bounds of the original prop_sub object using the function spsample. This new grid is converted to a spatial object by identifying the coordinates and then converting the data to a spatialpixel (gridded(grid)) and spatialgrid (fullgrid(grid)) object. Finally, the new spatialgrid object is projected to the same projection as the prop_sub object (proj4string(grid) <- proj4string(prop_sub)).


grid <- as.data.frame(spsample(prop_sub, "regular", n=5000))
plot(grid)

coordinates(grid) <- c("x1","x2")   
gridded(grid) <- T
fullgrid(grid) <- T
plot(grid)

proj4string(grid) <- proj4string(prop_sub)


####Next the spatialgrid object (grid) is filled with values using the idw function in the gstat package. The new interpolated data will be an estimation of the Price per Square Foot (pr_sq_ft) from the prop_sub object using a 2nd order exponent on the interpolation (idp=2.0).


pr_sq_ft_idw <- gstat::idw(pr_sq_ft~1, prop_sub, newdata=grid,
                           idp=2.0)
## [inverse distance weighted interpolation]


####The result of the interpolation is a new raster which we can create by loading the raster package and converting the IDW interpolated spatialgrid object. Using the raster package ulities we can also plot our new raster file and overlay the coorindates that were used to create the interpoaltion from the prop_sub object.


require(raster)
## Loading required package: raster
raster <- raster(pr_sq_ft_idw)
plot(raster)
points(prop_sub$x,prop_sub$y)
